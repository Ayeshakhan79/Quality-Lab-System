# Quality Lab System
This repository presents a sanitized case study of a Quality Lab System developed to support end-to-end batch testing workflows across Production, Lab, and OQ teams.  

**Note:** All screenshots are blurred and no confidential data, formulas, or app exports are included. This repository is intended for portfolio demonstration only.

## 1. Application Overview

The application flow has been **streamlined according to the requirements of Production, Lab, and OQ teams** to support an efficient batch testing workflow and improve traceability.

### Key Enhancements
- Streamlined task workflows, reducing manual tracking effort and improving operational efficiency.
- Enabled **real-time task visibility**, allowing users to clearly distinguish between pending, overdue, and completed tasks.
- Implemented **automated status transitions** to minimize human error and increase task tracking accuracy.
- Displayed both **due dates and completion dates** to support delay analysis and improve on-time task performance.
- Used consistent **visual indicators** to improve prioritization and clarity at a glance.
- Resolved recurring issues such as **status mislabeling, incomplete updates, and icon mismatches**, improving overall reliability and user confidence.

---

## 2. Dashboards

The solution includes **two dashboards** to provide real-time tracking and support quick decision-making:

1. **Finished Goods Dashboard**  
  Tracks sampling and testing status for finished product batches.

2. **Raw Material Dashboard**  
  Tracks sampling and testing status for incoming raw materials.

Both dashboards display **key KPIs, overdue tasks, capacity visualization**, and help stakeholders monitor progress effectively.

---

## 3. Tech Stack

### Dashboards
- **Data Source:** Azure Databricks  
- **Visualization:** Power BI

### Application
- **Front-End:** Power Apps  
- **Backend:** SharePoint
